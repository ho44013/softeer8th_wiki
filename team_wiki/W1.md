## 미션 2: SQL Tutorial

### sqlite3를 왜 사용하는가?

보통 SQL은 데이터베이스 서버(MySQL, PostgreSQL 등)에 접속해서 실행한다. 
하지만 SQLite는 데이터베이스 서버 없이 하나의 .db 파일만으로 데이터를 저장할 수 있다.
sqlite3는 Python과 SQLite 데이터베이스를 연결해 주는 인터페이스 역할.  

### sqlite3의 동작 과정

1. 데이터베이스 연결
2. Cursor 생성
    - Cursor는 SQL을 실행하는 객체
3. SQL 실행
4. 결과 가져오기
    - cur.fetchall()은 직전에 실행한 SQL의 결과를 전부 한 번에 가져오는 코드
5. Connection
    - 데이터베이스 연결
    - Commit
    - Rollback
    - Close

### 새로 알게 된/헷갈리는 구문

* Offset 
* AS [ ]
* SELECT INTO
* WHERE CustomerName LIKE '[bsp]%'; ---'[a-f]%';


## 미션 3: ETL 프로세스 구현하기

**wikipeida 페이지가 아닌, IMF 홈페이지에서 직접 데이터를 가져오는 방법은 없을까요?**

* IMF 사이트에서의 크롤링은 막혀있는 것을 확인했다. 해당 사이트를 직접 크롤링하는 것은 불가.
* 계속 갱신을 해주어야 한다는 단점도 있지만, GDP가 기록되어 있는 excel 파일도 해당 사이트에서 확인했다.
* IMF 사이트에서 제공하는 API가 있으므로, 해당 API를 활용해 필요한 정보를 가져와 구현하면 될 것 같다


**만약 데이터가 갱신되면 과거의 데이터는 어떻게 되어야 할까요? 
과거의 데이터를 조회하는 게 필요하다면 ETL 프로세스를 어떻게 변경해야 할까요?**

*  비즈니스 적으로 과거의 데이터가 필요하다면 저장을 해두되, 최근 몇 년간의 데이터만 저장하는 식으로 진행을 해야 할 것 같다.
* GDP를 기준으로 사업성을 평가하기 위함이기 때문에, 기본적으로 과거 데이터를 유지할 필요 없이 UPSERT 방식을 통해 기존 데이터를 업데이트 하면 될 것 같다.
  다만, 과거 데이터 조회가 필요한 상황이라면 데이터가 기록(업데이트)된 시점을 기입해 누적 관리해야 한다.
  가장 단순하게 생각한다면, 현재의 행에 반기별 컬럼을 추가해 관리하는것 (수평적 확장)도 그 방법이 될 것이다.
  만약, IMF가 임의로 규정한 Region에 대해서도 변동성을 보장하기 위해서는 Region별 국가 목록들에 대한 이력도 추가로 관리해야 한다. datetime column을 추가해 append 방식으로 적재한다.
*  기존 ETL 방식 대신 ELT 방식으로 변경한다. 
  원본 데이터를 먼저 데이터 웨어하우스에 적재한 뒤 필요한 변환을 수행하고, 새로운 데이터가 갱신될 때마다 기존 데이터를 덮어쓰지 않고 append 방식으로 지속적으로 저장한다.
  이를 통해 시점별 데이터 이력을 보존할 수 있으며, 과거 데이터와 최신 데이터를 모두 조회할 수 있다.
  → ELT 방식으로 변경한다면 데이터 웨어하우스가 아닌 Data Lake로 고려하는 것이 더 좋아보인다.
