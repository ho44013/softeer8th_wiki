## 2. 추가 논의 및 의사결정

### 2.1 문제 정의 재설정

* 기존: 긍정 리뷰 키워드(워드클라우드)로 신규 상품 컨셉을 결정한다.
* 변경: 오랜 굿즈 운영 경험상 일반적인 이슈에는 이미 대응 매뉴얼이 존재한다. 따라서 "기존 분류 체계(매뉴얼)에 잡히지 않는 새로운 고객 불편을 리뷰에서 발견하고, 상품군 단위로 원인을 특정해 개선 대안을 제시한다"로 문제를 다시 세운다.
* 뒷받침 근거

  * 집계 결과, 기존 aspect에 속하지 않는 신규 키워드가 약 30%를 차지한다면 → 아직 파악하지 못한 문제가 실재함.
  * 부정 키워드 집계에서도 베이비필로우가 최다 언급 → 분석 대상(베이비필로우) 선정의 타당성 확보.

### 2.2 리뷰 분류 방식: LLM 자동 분류 vs. 직접 검토

* LLM(Claude) 분류에 대한 우려: 블랙박스 특성상 어떤 추론 과정을 거쳐 분류했는지 보장할 수 없음 .
* "처리할 데이터가 10억 건이어도 같은 방법을 쓸 것인가?"라는 질문에 답할 수 있어야 LLM 사용이 정당화됨.
* 결정: 현재 리뷰 규모가 크지 않으므로 직접 원문을 검토한다. 키워드 라벨과 원문 의미가 어긋나는 사례(예: '하자·불량' 라벨이지만 실제 맥락은 하자·불량이 아님)가 있으므로 원문 대조가 필수. 대규모 확장 시의 전략(자동화 방안)은 예상 질문 답변으로 별도 준비한다.

### 2.3 Aspect 분류 체계

* 4대 aspect 설정: 하자·불량 / 가격 / 냄새 / 배송 → 이 축에서 해결 대안을 도출한다.
* 현재 방식(사전 기반 매칭): 도메인 지식으로 aspect별 키워드 리스트를 정의하고(예: 하자·불량 ← 부서짐 등 / 긍정 ← 귀여워·예뻐·캐릭터·디자인 / 냄새 ← 쉰내·담배냄새 등 세부 유형), 리뷰에 리스트 내 키워드가 하나라도 포함되면 해당 aspect로 매칭 후 집계한다. 회사가 오래 판매한 제품군은 주요 문제의 대분류가 이미 존재한다는 전제.
* 자동화 후보 리서치: ① 수동 사전 정의 ② 임베딩 + 시드 단어 확장(FastText) ③ LLM 기반 ABSA(리서치 기준 정확도 약 85%) 등 4가지 방법 중 1가지 선정. 현재 유력안은 LLM 기반 ABSA이며, 왜 우리 데이터셋에 적합한지 선정 사유를 문서화한다(피드백 3 대응).

### 2.4 지표 설계: Lift로 별점 영향도 측정

* 파이프라인: aspect 매칭 → 극성 분리 → 별점 그룹별 출현율 집계 → 출현율 ÷ 기준 출현율 = Lift
* 목적: 같은 키워드라도 별점 상향/하향에 실제로 얼마나 영향을 주는지 정량화 → 평점–텍스트 교차 분석(피드백 4)의 구현 수단.
* 보완: Lift 등 표의 각 컬럼에 해석 기준·설명을 1–2줄씩 반드시 추가한다. 해석 근거 없는 표 단독 제시 금지.
